## Overview

## Strategy 9-A: Protocol Insurance Fund Mining

### Mechanism
Perp DEX의 보험 기금(Insurance Fund)에 자본을 스테이킹하여 거버넌스 토큰 + 보험 기금 수익을 획득.

```
보험 기금 수익원:
├── 거래 수수료의 50%+ (프로토콜별)
├── 청산 수수료 (청산 시 패널티)
├── 펀딩레이트 잔여분
└── 거버넌스 토큰 보상

손실 발생 시 (우선순위):
1. 청산 수익 + 유기적 수익으로 커버
2. 프로토콜 보험 기금에서 충당
3. 스테이커 자금 사용 (최후 수단)
```

### Protocol Examples
| 프로토콜 | 메커니즘 | APY | 리스크 |
|---------|---------|-----|--------|
| DerivaDEX | DDX 토큰 마이닝 + 보험 수익 | 10~30% | 토큰 가치 하락 |
| dYdX | 보험 기금 스테이킹 | 5~15% | 보험 이벤트 발생 |
| GMX | GLP/GLV 풀 | 10~20% | LP 손실 |

### Return Profile
| 항목 | 수치 |
|------|------|
| 기본 APY (토큰 보상) | 10~30% |
| 토큰 가치 변동 | -50% ~ +200% |
| 보험 이벤트 빈도 | 연 1~3회 |
| 이벤트 시 손실 | 자본의 5~20% |
| **순 기대 수익 (토큰 홀드)** | **5~25%** |

### Korean Stock Application
```
Lighter DEX LLP Vault:
├── 한국 주식 Perp 거래 수수료 수취
├── 청산 수수료 수취
├── APY: ~60% (2025 초 데이터)
└── 리스크: 트레이더 수익 시 LP 손실

→ 한국 주식 거래량이 늘수록 Vault 수익 증가
→ 초기 유동성 제공자 = 높은 APY 가능
```

## Strategy 9-B: Impermanent Loss Hedging + Concentrated LP

### Mechanism
AMM(Automated Market Maker) 풀에 유동성을 제공하면서, Perp 숏으로 비영구적 손실(Impermanent Loss, IL)을 헤지.

### The IL Problem
```
IL 공식: IL = 2√(price_ratio) / (1 + price_ratio) - 1

가격 2배 변동 시: IL = -5.6%
가격 5배 변동 시: IL = -25.5%
가격 10배 변동 시: IL = -42.5%

→ 가격 변동이 클수록 LP 손실 증가
```

### Hedging with Perp
```
1. AMM 풀에 유동성 공급 (삼성 토큰/USDC)
2. 삼성 Perp 숏으로 IL 헤지
3. 헤지 비율: Delta 계산 기반 동적 조정
4. 수익 = 거래 수수료 - IL 잔여분 - 헤지 비용

Concentrated Liquidity (Uniswap V3 스타일):
├── 좁은 가격 범위 = 10x+ 수수료 수취
├── 넓은 가격 범위 = 낮은 수수료, 낮은 IL
└── 최적점: 예상 변동성에 맞춘 범위 설정
```

### Return Profile
| 항목 | 수치 |
|------|------|
| 총 수수료 수익 | 15~60% APY |
| IL 손실 (헤지 전) | -5~-25% |
| 헤지 비용 | -3~-8% |
| **순 기대 수익** | **7~30% APY** |

### Technical Requirements
- 동적 델타 계산 알고리즘
- 자동 리밸런싱 봇
- Gas 최적화 (빈번한 리밸런싱)
- 실시간 IL 모니터링

## Strategy 9-C: Tail Risk Hedging Sleeve (포트폴리오 보험)

### Mechanism
전체 차익거래 포트폴리오의 2~5%를 테일 리스크 헤지에 배분. 평상시에는 소폭 비용(-2%)이 발생하지만, 시장 붕괴(tail event) 시 **볼록한(convex) 수익**으로 전체 포트폴리오를 보호.

### Execution
```
Synthetic Put (합성 풋옵션):
1. Lighter KOSPI 200 Perp 소량 숏 (평상시 비용 발생)
2. 극단적 하락 시: 숏 이익이 기하급수적 증가
3. 포트폴리오 손실을 상쇄

구조:
├── 정상 시장: -2% 연간 비용 (보험료)
├── -10% 하락: +3% 수익
├── -20% 하락: +8% 수익
└── -40% 하락: +20% 수익 (포트폴리오 구원)
```

### Why Essential
```
Level 1~8 전략의 공통 취약점:
- 규제 급변 → 전 전략 동시 타격
- 시장 붕괴 → 청산 캐스케이드
- 오라클 장애 → 가격 오류 연쇄

→ Tail Risk Hedge가 "최후의 방어선" 역할
→ 비용은 다른 전략 수익으로 충분히 커버
```

### Return Profile
| 시장 상황 | 헤지 수익 | 포트폴리오 보호 |
|----------|---------|---------------|
| 정상 | -2% (비용) | 불필요 |
| 조정 (-10%) | +3% | 부분 보호 |
| 급락 (-20%) | +8% | 상당 보호 |
| 붕괴 (-40%+) | +20%+ | 완전 보호 |

### Edge Sustainability
**무한** — 테일 리스크는 구조적으로 존재하며, 시장이 합리적이어도 극단적 이벤트는 반복됨.
