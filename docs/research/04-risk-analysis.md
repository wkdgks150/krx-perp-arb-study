## Risk 1: Funding Rate Reversal (펀딩레이트 반전)

### Description
양(+) 펀딩레이트를 기반으로 현물 롱/Perp 숏 포지션을 유지하던 중, 시장 심리 변화로 펀딩레이트가 음(-)으로 전환되면 **역으로 지불**해야 한다.

### Trigger Conditions
- 시장 급락 → 숏 수요 폭증 → 음의 펀딩
- 매크로 이벤트 (금리 인상, 지정학적 리스크)
- 한국 특유: 반도체 업황 악화, 원화 급락

### Historical Precedent
- 2022 Terra/Luna 사태: BTC 펀딩 -0.1%/8h 지속 (연 -130%)
- 2024 Q4: 일시적 음의 펀딩 다수 발생

### Impact
| 전략 | 영향 |
|------|------|
| Strategy 1 (펀딩레이트) | **직접적** — 수익원 자체가 사라짐 |
| Strategy 2 (베이시스) | 간접적 — 스프레드 방향 변화 |
| Strategy 5 (델타뉴트럴) | **직접적** — 진입 조건 해제, 기존 포지션 손실 |

### Mitigation
- 펀딩레이트 모니터링 알람 설정 (임계값: 0.005%)
- 빠른 포지션 청산 메커니즘 구축
- 보수적 레버리지 (2~3x) 유지
- 역방향 전략 준비 (음의 펀딩 시 Perp 롱 + 현물 숏... 다만 공매도 제한)

## Risk 2: Liquidation Cascade (청산 캐스케이드)

### Description
급격한 가격 변동 시, 레버리지 포지션이 연쇄 청산되며 시장 충격을 증폭한다. 델타뉴트럴이라도 **한쪽 레그의 마진 부족**으로 강제 청산될 수 있다.

### Mechanism
```
1. 삼성 현물 +10% 급등
2. Lighter Perp 숏 포지션: -10% 미실현 손실
3. 마진 부족 → 청산 경고
4. 청산 시: Perp 숏 강제 커버 (매수) → 가격 추가 상승
5. 다른 숏 포지션도 연쇄 청산
6. 현물 롱은 이익이지만, 이미 Perp 숏은 청산됨
   → 델타뉴트럴 깨짐 → 방향성 리스크 노출
```

### Key Numbers
| 레버리지 | 청산까지 가격 변동 | 안전 여유 (권장) |
|---------|-------------------|----------------|
| 2x | 50% | 충분 |
| 5x | 20% | 보통 |
| 10x | 10% | 위험 (한국 주식도 10%+ 변동 가능) |
| 20x+ | 5% | 매우 위험 |

### Mitigation
- **레버리지 상한**: 2~3x (최대 5x)
- **마진 버퍼**: 유지 마진의 150% 이상 유지
- 자동 마진 추가(auto-deleverage) 메커니즘 활용
- 가격 알람: 유지 마진율 150%에서 경고

## Risk 3: Oracle Manipulation / Latency (오라클 조작/지연)

### Description
Perp DEX의 가격은 오라클에 의존한다. 오라클 피드의 조작, 지연, 장애는 잘못된 가격으로의 청산이나 부당한 포지션 생성을 초래할 수 있다.

### Attack Vectors
1. **플래시론 공격**: 대규모 일시적 가격 조작
2. **오라클 지연**: 급변 시장에서 가격 업데이트 지연 (→ 차익거래자에게 착취당함)
3. **단일 오라클 의존**: 단일 실패점

### Latency Comparison
| 오라클 | 업데이트 간격 |
|--------|-------------|
| Chainlink Data Streams | ~1초 |
| Stork Network | ~500ms |
| Pyth (Solana) | ~400ms |
| Lazer (신규) | ~1ms |

### KRX Off-Hours 특수 리스크
```
KRX 폐장 중 (15:30~09:00):
- 현물 가격 동결 → 오라클이 참조할 최신 가격 없음
- Perp 가격은 글로벌 요인으로 변동
- 오라클 정확도 하락 가능
- Mark Price와 Oracle Price 괴리 확대
```

### Mitigation
- 복수 오라클 교차 검증 (Chainlink + Stork + 자체 가격 피드)
- 오라클 장애 시 포지션 자동 축소
- 프로토콜 수준 지연 메커니즘 (주문~체결 사이 가격 업데이트)
- KRX 폐장 시간대 레버리지 축소

## Risk 4: Smart Contract Risk (스마트컨트랙트 리스크)

### Description
Perp DEX는 복잡한 스마트컨트랙트 로직(마진 계산, 청산 로직, 펀딩레이트 정산)을 포함한다. 코드 결함은 자금 손실로 이어질 수 있다.

### Historical Incidents
- 2023: DEX 해킹으로 $200M+ 손실 (업계 합산)
- Oracle 조작을 통한 마진 계산 오류 익스플로잇 다수

### Vulnerability Types
1. **마진 계산 오류**: 미세한 로직 결함 → 잘못된 포지션 밸류
2. **청산 로직 결함**: 조기/미청산
3. **검증 누락**: 무효한 거래 실행
4. **정밀도 오류**: 레버리지 계산 반올림 이슈

### Lighter DEX Specific
- ZK-rollup 기반 → 추가적인 복잡성 (ZK-proof 관련 버그 가능)
- 감사 이력: 확인 필요
- 보험/보상 메커니즘: 확인 필요

### Mitigation
- 감사(audit) 이력 확인된 프로토콜만 사용
- DeFi 보험 (Nexus Mutual 등) 가입 검토
- 포지션 규모 분산 (단일 프로토콜에 전체 자본 집중 금지)
- 신규 프로토콜 진입 시 소액으로 테스트

## Risk 5: Regulatory Risk (규제 리스크)

### Description
한국과 글로벌 규제 환경이 RWA perp-DEX 차익거래의 합법성과 실현가능성을 근본적으로 변경할 수 있다.

### Korean Regulatory Risks
| 리스크 | 확률 | 영향 |
|--------|------|------|
| 해외 DEX 이용 제한 강화 | 중 | 전략 전체 불가 |
| 스테이블코인 규제 시행 | 고 | 자본 이동 제한 |
| 가상자산 소득세 시행 (2027) | 고 | 순수익 22% 감소 |
| 외환거래법 집행 강화 | 중 | 법적 리스크 증가 |
| 토큰화 증권 프레임워크 시행 | 고 | 새 기회 + 새 규제 |

### Global Regulatory Risks
| 리스크 | 확률 | 영향 |
|--------|------|------|
| SEC의 주식 perp 단속 | 중 | 플랫폼 폐쇄/접근 제한 |
| EU MiCA의 RWA 규제 확대 | 중 | 유럽 유동성 감소 |
| 글로벌 AML 강화 | 고 | KYC 요구 확대 |

### Mitigation
- FSC/금융감독원 발표 지속 모니터링
- 법률 자문 확보 (크립토 + 외환거래법 전문)
- 규제 변화 시 신속 포지션 정리 계획
- 복수 관할권 분산 (싱가포르, 두바이 등)

## Risk 6: Execution / Slippage Risk (실행/슬리피지 리스크)

### Description
차익거래는 양 레그의 동시 실행이 핵심이다. 한쪽 실행 지연이나 가격 변동은 수익을 잠식한다.

### Sources of Slippage
| 원인 | DEX | KRX |
|------|-----|-----|
| 유동성 부족 | 큰 주문 시 높은 슬리피지 | 소형주 유동성 부족 |
| 블록 확인 시간 | ZK-rollup ~수초 | N/A |
| API 지연 | RPC 노드 지연 | 증권사 API 지연 |
| 가스비 급등 | 네트워크 혼잡 시 | N/A |

### Quantification
| 항목 | 예상 범위 |
|------|----------|
| DEX 슬리피지 | 0.05~0.5% (주문 크기에 비례) |
| 양 레그 시간차 | 1초~수 분 |
| 가스비 | $0.1~$10 (네트워크별) |
| 증권사 수수료 | 0.015~0.03% |

### Mitigation
- 유동성 깊은 시간대에 실행 (KRX 개장 30분 후~폐장 1시간 전)
- 주문 분할 (TWAP/VWAP 알고리즘)
- 빠른 RPC 노드 + 전용 연결
- 최소 수익 임계값 설정 (슬리피지 초과 시 실행 중단)

## Risk 7: Liquidity Risk (유동성 리스크)

### Description
한국 주식 perp의 DEX 유동성은 BTC/ETH 대비 극히 제한적이다. 대규모 포지션 진입/청산 시 시장에 과도한 영향을 미친다.

### Estimated Liquidity (Lighter DEX Korean Stocks)
| 자산 | 예상 일일 거래량 | 최대 포지션 (영향 < 0.5%) |
|------|-----------------|-------------------------|
| Samsung perp | 추정 필요 | 추정 필요 |
| SK Hynix perp | 추정 필요 | 추정 필요 |
| KOSPI 200 perp | 추정 필요 | 추정 필요 |

> Note: Lighter DEX의 한국 주식별 구체적 거래량 데이터는 추가 조사 필요

### Mitigation
- 포지션 규모를 일일 거래량의 1~3% 이내로 제한
- 진입/청산 분할 실행
- 유동성 깊은 자산(Samsung, KOSPI 200) 우선 활용
- 급격한 유동성 소실 시 자동 포지션 축소

## Risk Summary Matrix

| Risk | 확률 | 영향 | 관련 전략 | 완화 가능성 |
|------|------|------|----------|------------|
| 1. 펀딩레이트 반전 | 중 | 고 | 1, 5 | 모니터링 + 빠른 청산 |
| 2. 청산 캐스케이드 | 저~중 | 매우 고 | 1~5 전체 | 보수적 레버리지 |
| 3. 오라클 조작/지연 | 저 | 고 | 1~5 전체 | 복수 오라클 |
| 4. 스마트컨트랙트 | 저 | 매우 고 | 1~5 전체 | 감사 확인 + 분산 |
| 5. 규제 | 중~고 | 매우 고 | 1~5 전체 | 법률 자문 + 모니터링 |
| 6. 실행/슬리피지 | 중 | 중 | 1~3 | 알고리즘 + 분할 실행 |
| 7. 유동성 | 중~고 | 중 | 1~5 전체 | 포지션 규모 제한 |

### Overall Risk Assessment
**"Risk-Free"는 학술적 정의상 불가능하다.** 위 7개 리스크 중 어느 하나도 완전히 제거할 수 없으며, 특히 규제 리스크와 자본통제는 한국 거주자에게 근본적 제약이다. 현실적으로는 **"Low-Risk, Market-Neutral"** 전략으로 접근해야 한다.
